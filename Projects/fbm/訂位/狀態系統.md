## 概述

訂位系統使用四個獨立的狀態維度，交叉決定卡片上的圖示、文字、操作按鈕。此文件是所有卡片元件的共用參考。

## 四維狀態定義

### 訂位類型 (reservationType)

| Code | 名稱 | 說明 | 來源 |
|------|------|------|------|
| RSV | 訂位 | 一般預約訂位（電話/線上/PMS） | 建立時指定 |
| WLB | 候補 | 訂位候補，可轉正為 RSV | 建立時指定 |
| WIO | 候位 | 現場排隊候位 | 建立時指定 |
| WLK | 現場客 | 未預約直接到場 | 建立時指定 |

### 訂位狀態 (reservationStatus)

| Code | 名稱 | 說明 |
|------|------|------|
| CNF | 已確認 | 訂位已確認，正常狀態 |
| CXL | 已取消 | 訂位已取消（需看 cancelReason 判斷原因） |
| WTC | 候補轉正中 | 候補準備轉為正式訂位（可點擊圖示觸發） |
| WIO | 候位中 | 候位狀態 |
| WLB | 候補中 | 候補狀態 |

### 現場狀態 (onsiteStatus)

| Code | 名稱 | 說明 |
|------|------|------|
| NST | 未入座 | 客人尚未入座（或 null） |
| STD | 已入座 | 客人已入座，記錄入座時間 |
| LFT | 已離席 | 客人已離席 |
| CXL | 已取消 | 現場取消 |

### 通知狀態 (reminderStatus)

| Code | 名稱 | 說明 |
|------|------|------|
| NNF | 未通知 | 尚未發送通知 |
| NDF | 通知失敗 | 通知發送失敗 |
| NTF | 已通知 | 通知已成功送達 |
| HLD | 客人保留 | 客人已確認保留此訂位 |

### 取消原因 (cancelReason)

| Code | 名稱 | 說明 |
|------|------|------|
| NSW | No Show | 客人未到場 |
| OTH | 其他 | 其他原因 |
| NDC | 未確認 | 未確認取消 |
| WTO | 候位逾時 | 候位超時自動取消 |

### 訂金狀態 (depositStatus)

| Code | 名稱 | 說明 |
|------|------|------|
| PDP | 已付 | 訂金已收取 |
| NDP | 未付 | 訂金尚未收取 |

### 訂位來源 (reservationSource)

| Code | 名稱 | 說明 |
|------|------|------|
| ONL | 線上 | 線上預訂 |
| TEL | 電話 | 電話預約 |
| WLK | 現場 | 現場登記 |
| STF | 服務生 | 服務生代建 |
| CRS | 訂位中心 | 預約中心建立 |
| PMS | PMS | 飯店管理系統 |
| POS | POS | POS 系統 |

## StatusIconRender 優先序

StatusIconRender 是所有卡片共用的狀態圖示元件。它依照**嚴格的優先順序**判定，命中第一個就停止。

```
判定開始
  │
  ├─ reservationStatus = WTC ?
  │  └─ YES → 🟢 WaitingToReservationIcon（綠色箭頭，可點擊觸發轉正）
  │
  ├─ type = WIO 且 status = CXL ?
  │  └─ YES → ⚪ CancelIcon（灰色取消，不可點擊）
  │
  ├─ status = CXL ?
  │  ├─ cancelReason = NSW ?
  │  │  └─ YES → ⚪ NoShowIcon（灰色 No Show）
  │  ├─ endTime 在未來（未過期）?
  │  │  └─ YES → 🔴 RestoreIcon（紅色復位，可點擊觸發復位）
  │  └─ else → ⚪ CancelIcon（灰色取消）
  │
  ├─ 任一桌位 onsiteStatus = STD ?
  │  └─ YES → 🟠 SeatedIcon（橘色椅子 + 入座時間 HH:mm）
  │
  ├─ 任一桌位 onsiteStatus = LFT ?
  │  └─ YES → ⚪ LeftIcon（灰色門）
  │
  ├─ reminderStatus = NDF ?
  │  └─ YES → ⚪ NotifiedErrorIcon（灰色錯誤）
  │
  ├─ reminderStatus = HLD ?
  │  └─ YES → 🟢 ReservationIcon（綠色勾人）
  │
  ├─ reminderStatus = NTF ?
  │  └─ YES → 🟢 SendMessageIcon（綠色勾 + 通知時間 HH:mm）
  │
  ├─ reminderStatus = NNF ?
  │  └─ YES → 🟢 UnNotifiedIcon（綠色待辦勾）
  │
  └─ else → 空白
```

### 可互動的狀態圖示

只有兩個圖示是可點擊的：

| 圖示 | 動作 | API 操作 |
|------|------|----------|
| WaitingToReservationIcon | 候補轉正 | `WLB_TO_CNF` |
| RestoreIcon | 取消復位 | `CANCEL_RESTORE` |

點擊時會先執行 `onEdtingCheck`（確認是否有未儲存編輯），通過後才呼叫 API 更新狀態。
