# 套件使用說明

從業務專案（crs、pms、fbm…）引入 `vue-bac-lib` 套件的實際寫法。
以 crs 專案為參考基準。

> 與 [[Model實作指南]] 的差異：本文以「套件使用者」視角撰寫，import 路徑、命名慣例皆以實際業務專案為準。

## Import 方式

所有 API 從 `vue-bac-lib` 頂層引入，factory 統一透過 `modelFactory` 物件存取：

```ts
import {
  modelFactory,           // 所有 factory 的入口物件
  getGenLayout,           // 搜尋欄位 layout helper
  getGridLayout,          // Grid 欄位 layout helper
  getGenToolbarLayout,    // Toolbar layout helper
  useDialog,              // Dialog 管理
  useTools,               // 工具（toast、alert、i18n）
  isValidationField,      // 驗證工具
  searchConditionToQuery, // 搜尋條件轉 query string
  // UI 元件
  BacTextBox, BacDropdownlist, BacButton, BacWithOperator,
  GridTextBox, GridDropdownList, GridDateTimePicker,
  // ...
} from 'vue-bac-lib'
```

```
app/pages/[aid]/
└── CRS0110010.vue     ← 頁面進入點，只負責掛載元件
```

## 頁面進入點（.vue）

頁面本身非常薄，只負責把 program 的 factory 包成 Vue 元件：

```vue
<!-- app/pages/[aid]/CRS0110010.vue -->
<script lang="ts">
import factory from '../../program/CRS0110010'
import { componentFactory } from '@/utils/helper'

export default componentFactory(factory)
</script>
```

`componentFactory` 是包裝 `modelFactory.componentFactory` 的本地 helper：

```ts
// app/utils/helper.ts
import { modelFactory, type IBaseCore } from 'vue-bac-lib'

export function componentFactory(baseCore: () => { main: IBaseCore }) {
  return modelFactory.componentFactory(baseCore)
}
```
